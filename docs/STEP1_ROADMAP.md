# Step 2+ Roadmap

## Priority sequence

### 1) Specification finalization
- Lock a primary cutoff/event-window definition and document alternatives.
- Finalize baseline control sets for first and second stages.
- Define the canonical estimation sample with explicit inclusion/exclusion logic.

### 2) Identification stress tests
- Add placebo cutoff tests.
- Run alternative bandwidth selectors and local polynomial specifications.
- Verify robustness to calendar composition and availability-based sample rules.

### 3) Output standardization
- Export clean regression tables for baseline + robustness specs.
- Export figure set (price trends, discontinuity visuals, coefficient-by-bandwidth).
- Add reproducibility instructions for one-command reruns where feasible.

### 4) Heterogeneity and mechanism checks
- Split effects by host scale, tenure, and booking flexibility settings.
- Examine whether effects are concentrated in segments most likely to adopt algorithmic recommendations.

### 5) External validity and expansion
- Port cleaned pipeline and main specification to additional cities.
- Compare city-level effect patterns and discuss competition-policy implications.

## Deliverables target
- A methods-complete empirical draft with reproducible code paths, baseline/robustness tables, and clearly stated antitrust interpretation boundaries.
